# Skill Versioning Guidelines

## Overview

Skills use Semantic Versioning (SemVer) to track changes and enable proper dependency management.

## Version Format

```
MAJOR.MINOR.PATCH
```

- **MAJOR**: Breaking changes that are not backward compatible
- **MINOR**: New features that are backward compatible
- **PATCH**: Bug fixes that are backward compatible

## Version Field

Add `version` field to SKILL.md frontmatter:

```yaml
---
name: my-skill
version: "1.2.0"
description: Skill description...
---
```

## When to Bump Version

### MAJOR (X.0.0)
- Remove or rename required fields
- Change skill behavior in ways that break existing workflows
- Change required dependencies that existing users have
- Remove scripts or references that were previously required

### MINOR (1.X.0)
- Add new optional features
- Add new scripts or references
- Improve documentation
- Add new capabilities while maintaining backward compatibility

### PATCH (1.2.X)
- Fix bugs in documentation
- Fix bugs in scripts
- Improve existing functionality without changing behavior
- Update dependencies to patch versions

## Best Practices

1. **Always start new skills at 1.0.0**
2. **Update version when making any changes** to existing skills
3. **Document breaking changes** in skill description or changelog
4. **Use index.json** to track all skill versions across the repository

## Auto-Discovery

The `skills/index.json` file is auto-generated by GitHub Actions when SKILL.md files change. This enables:
- Skill discovery without manual cataloging
- Version tracking across all skills
- Automated tooling support

## Index.json Schema

```json
{
  "generated_at": "2026-01-01T00:00:00Z",
  "total_skills": 32,
  "skills": [
    {
      "name": "skill-name",
      "version": "1.0.0",
      "description": "Skill description",
      "category": "system|curated|experimental",
      "path": ".system/skill-name",
      "display_name": "Skill Name",
      "short_description": "Short description",
      "default_prompt": "Default prompt"
    }
  ]
}
```

## Migration Notes

When adding version to existing skills:
- Default to "1.0.0" for any skill without a version field
- This represents the initial version before tracking began
