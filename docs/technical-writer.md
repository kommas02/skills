# Technical Writer Domain

## Domain Scope

The technical-writer domain focuses on:
- Documentation creation and maintenance
- Architectural constitution (blueprint.md)
- Naming conventions and consistency
- README and contribution guidelines
- API documentation standards

## Common Tasks

- Creating and updating documentation files (README.md, docs/*.md)
- Maintaining architectural blueprints
- Ensuring naming consistency across repository
- Linking documentation appropriately
- Creating contribution guidelines

## Issue Detection

Technical-writer issues typically have:
- Label: `documentation`
- Owner Agent: `Technical-Writer`
- Problem: Missing docs, inconsistent naming, broken links

## Key Files to Monitor

- `docs/blueprint.md` - Architectural constitution
- `README.md` - Main documentation entry point
- `contributing.md` - Contribution guidelines
- Skills directories - SKILL.md files

## Collaboration

- Work with DX-Engineer on naming conventions
- Coordinate with architect on structural changes
- Support other agents with documentation needs

## Best Practices

1. Always link new documentation from README.md
2. Use clear, concise language
3. Include examples where helpful
4. Follow existing documentation patterns
5. Validate links before submitting PR
