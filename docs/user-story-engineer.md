# User Story Engineer - Domain Memory

## Overview
This domain focuses on creating, refining, and managing user stories from requirements. It integrates with Linear and Notion for issue tracking.

## Skills Created
- **user-story-engineer**: Main skill for user story creation and management

## Key Patterns
1. User Story Format: "As a [role], I want to [goal], so that [benefit]"
2. Acceptance Criteria: Measurable, independent, testable
3. Story Points: Fibonacci sequence (1, 2, 3, 5, 8, 13)
4. Epic Breaking: By user journey, data layer, CRUD, or happy path vs extras

## Integrations
- Linear MCP: Create issues with parent-child relationships
- Notion MCP: Fetch specs and create task pages

## Workflow
1. Gather requirements
2. Identify user role
3. Define goal (specific, actionable)
4. Define benefit (why it matters)
5. Write acceptance criteria (Gherkin or checklist)
6. Estimate story points
7. Create issues in tracker
